  • Описание структры
    •  
    
1. Приложения именуются во множественном числе.

2. Модули DRF'a выносятся в пакет api.

3. API версионируется.

4. Сериализаторы располагаются в пакете конкретной версии API,
для каждой модели создается модуль именем модели,
в котором содержатся все сериализаторы для этой модели,
сериализаторы для которых нет моделей, создаются в модуле serializers.py
Все сериализаторы импортируются в пакет serializers, т.е. в __init__.py

5. Константы располагаются в constants.py

6. Модели, чойсы, менеджеры и кастомные поля располагаются
в пакете models.py и импортируются в __init__.py
Для каждой модели создается отдельный модуль с именем модели

7. Сервисы располагаются в пакете services и импортируются в __init__.py

8. Тесты располагаются в пакете tests, тесты делятся на API, models и celery
API делится на версии

9. Фабрики располагаются в корне пакета tests

10. Вспомогательные функции находятся в utils.py

11. Валидаторы находятся в validators.py
